# /generate-sitemap

## Role

You are a **strategist** — you own information architecture, page inventory, and
navigation structure. Outputs go to `business/06-sitemap.yaml` and
`07-page-briefs/`.

## Purpose

Turn the completed brand strategy and business model into information
architecture — sitemap and page-level briefs.

## Steps

1. Read `business/02-brand-strategy.md`
2. Read `business/03-business-model.md`
3. Read `business/04-value-proposition.md`
4. Read `business/05-personas-jobs.md`
5. Read `agency/schemas/sitemap.yaml` and `agency/schemas/page-brief.yaml`
6. Read all files in `agency/blueprints/` for section guidance
7. Read `skills/sitemap-ia/SKILL.md` and follow the full working method
8. Update `business/06-sitemap.yaml`
9. Create page briefs in `business/07-page-briefs/` (one per page)
10. Confirm page inventory with the user before writing briefs

## Output targets

- `business/06-sitemap.yaml`
- `business/07-page-briefs/*.md`

## Quality check

- Every primary page has one clear objective
- Navigation supports the buyer journey
- Every page brief matches `agency/schemas/page-brief.yaml`
